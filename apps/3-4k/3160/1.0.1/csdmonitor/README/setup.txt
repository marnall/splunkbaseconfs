# ClearSky Data Monitoring App for Splunk Enterprise (csdmonitor) setup instructions 
# v1.0

ClearSky Data Monitoring App (csdmonitor) is a Splunk application that monitors status and logs of the attached ClearSky storage. The application includes a modular input (ClearSky Event Collector) that is using REST APIs to collect logs and stats from ClearSky storage. 

In order to collect data, the REST API endpoint must be configured. The configuration screen can be launched from the csdmonitor app by clicking on the Configuration item in apps menu. As an alternative, it can be accessed from the top level Splunk menu (Settings -> Data Inputs tools).  

ClearSky Data Monitoring App can collect information from multiple ClearSky storage pods. When configuring endpoints, make sure that they are not pointing to the same storage pod, otherwise duplicate information will be displayed in the app. 

The following parameters must be configured for the endpoint in order to collect data:
- Storage pod name: Name of storage pod (preferred) or other endpoint identifier. This name will identify  this storage pod in the app and must be unique. Note that this name cannot be changed later.  
- Edge cache IP: Virtual IP address (client IP) of the ClearSky storage as reported by ClearSky Storage Manager. Spunk will be accessing ClearSky REST APIs on this IP so it must be accessible from the Splunk server.
- ClearSky customer ID: Your customer ID string, e.g. 1000D or 10005 
- Authentication user: ClearSky login username. It is recommended to create a read-only ClearSky account for using by this application. This application requires only read-only access to ClearSky management.   
- Authentication password: The corresponding login password.

Additional parameters can be configured in the advanced section:
- Polling interval: Frequency of collecting data. Default value is 60 seconds.
- Back-off time: Wait time before retry after timeout or error. Default is 60 sec.

After configuring the endpoint, you can return to the application. If connection was configured properly, you should see screens populated with data in 1-2 minutes. If you don’t see any data displayed, open the ClearSky Event CollectorLog tool in the app’s Advanced menu and check application logs. They should indicate if there is any error connecting to the storage.
