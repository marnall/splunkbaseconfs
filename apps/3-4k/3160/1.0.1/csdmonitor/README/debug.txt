# ClearSky Data Monitor modular input debug instructions
# v1.0

ClearSky modular script (ClearSky Event Collector) is a python script (csd.py) located in the app’s bin directory. Script has a few dependencies from other Python libraries. All of them are included into app’s /bin directory.

When running, script is pushing logs via stderr to the standard Splunk’s log file splunk.log located in $SPLUNK_HOME/var/log directory. To see logged events, search the file for the “/bin/csd.py” signature. Note that DEBUG messages are not enabled by default. To enable debug output, start Splunk process with the —debug switch, or alternatively modify Splunk logger configuration files. You can also enable debug settings from Splunk Web Settings menu.

In addition, the ClearSky Monitoring app itself has a pre-built view located from the app’s Advanced menu (Event Collector Log). This view makes a query to splunkd.log and pulls all csd.py messages from there. You can use it to quickly review modular script logs.       

Sometimes it is necessary to run the modular script outside of the Splunk environment to debug and see all events going to Splunk. You can use the Splunk’s command line splunkd command to run the script in stand-alone mode. Provide script name and input name as arguments. There are a few commands available:  

Print XML configuration this script receives from Splunk:
$SPLUNK_HOME/bin/splunk cmd splunkd print-modinput-config --debug csd csd://<input>

Run data collection and output events and logs to console:
$SPLUNK_HOME/bin/splunk cmd splunkd print-modinput-config --debug csd csd://<input> | /
$SPLUNK_HOME/bin/splunk cmd python $SPLUNK_HOME/etc/apps/csdmonitor/bin/csd.py

Note that if you are running script in stand-alone mode, it is not actually pushing any data to Splunk. Also, it cannot make queries back to Splunk (no session token available) so checkpointing will not be working (you will see Authentication exception in the script output when it tries to run the query)
