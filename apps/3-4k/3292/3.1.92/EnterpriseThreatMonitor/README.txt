Enterprise Threat Monitor 3.1 for SAP Applications by ESNC GmbH, Germany
===========
Enterprise Threat Monitor (ETM) agent collects security logs from SAP® applications and uses its built in attack detection signatures to precorrelate this information. Resulting high quality events are sent to Splunk. Enterprise Threat Monitor App, which is installed on Splunk can be used for displaying the results and for further processing the events. 

ENTERPRISE THREAT MONITOR AGENT INSTALLATION
===========
Please follow the instructions at https://www.enterprise-threat-monitor.com/installation for installation requirements and configuration for retrieving events from your SAP applications.

PROCESSING EVENTS SENT TO SPLUNK
===========
Enterprise Threat Monitor Splunk app uses Splunk's HTTP event collector for receiving the events. The received events are collected on an index specific to Enterprise Threat Monitor. You can use the HTTP Event Collector management page on your Splunk web interface to configure an access token specific to Enterprise Threat Monitor. 

SECURITY
===========
Please make sure Splunk is configured for HTTPS. For security only HTTPS connections are allowed from Enterprise Threat Monitor agent to Splunk HTTP Event Collector. 

Please create an index specific to Enterprise Threat Monitor and configure the Splunk HTTPs Event Collector endpoint you created for ETM to accept data to this index only.

The Splunk HTTP Event Collector token, which you created, must only be used by ETM. It should not be shared with other applications forwarding data to Splunk.

SUPPORT
===========
Please refer to https://www.enterprise-threat-monitor.com/frequently-asked-questions for troubleshooting Enterprise Threat Monitor. 

Support is available via support-siem@esnc.de during weekdays excluding public holidays.
