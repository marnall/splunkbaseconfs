DataKeep Security Intelligence
-------------------------------

Description

    The DataKeep Security Intelligence application for 
	Splunk (R) provides a number of security intelligence reports and enables 
	search operations that can be used to extract information from the event 
	logs produced by DataKeep encryption agents and the DataKeep Policy, 
    Provisioning, and Management console (PPM).

    See https://securityfirstcorp.com/datakeep/

    Most of the required information is extracted from log messages 
    generated by DataKeep agents when programs access files in directories 
	protected by a DataKeep File agent or Volume with Policy agent.  
    The agent has to be configured to protect selected directories and 
	control access to the files in the protected directories based on
	what users/groups are permitted or denied access.

    * This app by default assumes log data will be received by Splunk 
      via TCP on port 9998.
	  
    * The log streams provided to Splunk must be in Syslog LEEF format, 
      else field extraction will not work correctly. This is selected in
      the PPM when defining the syslog forwarding formats.

    * The reports and searches might not provide useful information
      until a sufficient volume of log events has been captured by Splunk.
