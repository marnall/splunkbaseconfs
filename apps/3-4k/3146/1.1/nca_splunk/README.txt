NCA-Splunk App
==============

This app extends Splunk to provide integration with Novetta Cyber Analytics (NCA).


Installation
------------

Installing the app into your Splunk environment requires extracting the nca-splunk
directory to the etc/apps/ directory under your Splunk base directory (typically 
/opt/splunk). This add-on should be installed to each instance of Splunk,
especially the forwarders which will index results files from NCA.

Once the app is installed, you will need to configure the workflow action to point
to your NCA web interface. This can be done by logging into Splunk with admin or
power user privileges, navigate to Settings -> Fields -> Workflow Actions. Locate
the NCASessionsQuery_field_menu workflow action and click on it to edit it. In the
URI box under "Link configuration" change NCA_HOST to the appropriate hostname
(and port, if applicable) for your NCA web interface. Also confirm that the
protocol (http or https) is correctly specified. When finished, click the Save
button.

You will also want to ensure that Splunk is configured to ingest the result CSVs
being generated by NCA. The directory to which NCA will write these files can be
configured in NCA's nfaweb/settings.py file. Typically it is /mnt/nfaweb/splunk.

Make sure that splunk is configured to ingest the files generated by NCA using
the source type 'novetta-csv-results'. This can be configured in
$SPLUNK_HOME/etc/system/local/inputs.conf.

See: http://docs.splunk.com/Documentation/Splunk/6.0.2/Data/Bypassautomaticsourcetypeassignment

Finally, on the NCA side make sure that SPLUNK_URL is correctly specified in your
/etc/noesis.conf file.
