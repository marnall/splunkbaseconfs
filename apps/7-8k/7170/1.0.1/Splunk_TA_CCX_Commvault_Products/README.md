**Description:**

The CCX Add-On for Commvault Products looks to provide a single field extraction bundle for Commvault log types.
This TA was built using a large dataset and endeavours to be the most CIM compliant comprehensive field extraction TA avaliable for;
- Commvault Backup - Syslog

Fully compatible with Splunk Enterprise and Splunk Cloud, built by an Ops team for Ops teams.

**Features:**

- This TA currently supports logtypes tagged under the following CIM datamodels: Alerts, Change, and Authentication.

**Compatibility:** 

| Splunk Enterprise versions | 10, 9.4, 9.3, 9.2, 9.1 |
| --- | --- |
| CIM | 6.x 5.x |
| Platforms | Platform independent |
| Vendor Products | Commvault |
| Service Provider | CyberCX |

**Requirements:**
- This Add-On is intended to be installed on Heavy Forwarders, Indexers, and Search Heads.

**Installation:**
- This Add-On is intended to be installed on Heavy Forwarders, Indexers, and Search Heads.
- Is recommended to use separate index per Commvault products.

**Known issues:**
- Currently this Add-on supports Commvault Backup logs ingested via Syslog only.
