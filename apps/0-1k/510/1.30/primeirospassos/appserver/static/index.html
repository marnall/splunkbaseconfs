Splunk - Primeiros Passos

Splunk é a ferramenta de buscas de TI. Splunk permite que você indexe, busque, crie alertas e relatórios sobre qualquer informação de TI, em tempo real, para o gerenciamento de aplicações, operações de TI, segurança, compliance e mais. Splunk pode digerir qualquer tipo de informação: Arquivos de Log, métricas de sistema, configurações e mais.

Escolha um tópico abaixo para iniciar.

Aonde estou agora?

Splunk é feito de Aplicações (Apps). Apps criam diferentes contextos para suas informações utilizando grupos de visualizações (views), painéis de controle (dashboards) e configurações (configurations). Neste momento, você está na app Primeiros Passos, mas existem várias outras aplicações disponíveis para você. Por exemplo, Splunk vem com uma app que foi especialmente desenhada para funcionar com o Sistema Operacional que você está usando. Sua implementação Splunk pode também incluir apps customizadas ou pré-instaladas. Por padrão, Splunk inclui a app Search (Busca), e todos os exemplos apresentados aqui vão abrir a app Search em uma nova janela (ou tab, se você estiver usando o Firefox).

Indexar

Se um equipamento pode gerar, Splunk pode indexar. Sim, é isso mesmo -- Splunk pode indexar qualquer informação de TI (não importa qual a origem ou o formato) sem a necessidade de tradutores, conectores ou adaptadores. Alimente Splunk com qualquer informação desde syslogs de servidores Unix até Event Logs de Windows, passando por logs de aplicações customizadas e métricas de sistema -- você finalmente terá visibilidade real de toda a sua operação.

Imaginando o que é informação de TI? Leia mais aqui.

Neste servidor Splunk

Splunk pode indexar informação local, de forma contínua ou pontual. Configure Splunk para indexar um arquivo ou diretório através do Splunk Manager.

  1. Clique Manager » Data inputs » Files & Directories.
  2. Clique New.
    3. Me leve até lá...
  3. Escolha ''Monitor a file or directory'' para indexar informação de forma contínua.
  4. Escolha ''Index a file on the Splunk server'' para indexar um arquivo uma vez.
  5. Escolha ''Upload a local file'' para fazer o upload de um arquivo que se encontra em sua máquina cliente.
  6. Defina outras opções seguindo as instruções no restante da página. Por exemplo, defina um nome de host (origem), sourcetype (tipo de informação) ou index (índice). Ou você pode deixar a maioria das opções no padrão (default) e Splunk vai cuidar disso para você.

Em uma máquina Windows remota

Use Splunk para colher informações do Windows de outras máquinas em sua rede. Você tem algumas opções, dependendo de como sua rede está configurada:

  1. Se está executando Splunk em uma máquina Windows, você pode utilizar WMI para coletar logs de eventos Windows, bem como coleções Eventlog, monitoria de registro e de Active Directory. Todas as suas opções estão listadas em Manager » Data inputs.
    2. Me leve até lá...
  2. Se está executando Splunk em uma máquina não-Windows, você pode utilizar um agente para enviar a informação da máquina Windows para Splunk. Configure um forwarder (encaminhador) Splunk na máquina remota. Para mais informações sobre forwarders, verifique o tópico ${_("Splunk forwarders")} no Manual de Administração.
  3. Ou, você pode sempre indexar informações de diretórios que estão compartilhados em sua rede. Simplesmente navegue até a página Data Inputs do Splunk Manager e escolha ''Monitor a file or directory''. Digite o caminho completo até o arquivo ou diretório. Por exemplo, no Windows você pode colocar: c:\apache\apache.error.log para monitorar um arquivo local, ou \\hostname\apache\apache.error.log para monitorar um arquivo em uma máquina Windows remota.
    4. Me leve até lá...

Configure entradas (inputs) para uma app

Por padrão, Splunk traz uma app escrita especificamente para o seu Sistema Operacional (OS), tais como Windows ou *nix. Estas apps permitem a você monitorar performance do sistema, logs de eventos e mudanças de registro e filesystem. Configure entradas para estas apps ao ativá-las. Você pode também fazer o download de mais apps na SplunkBase para se adequar às informações do seu ambiente, como por exemplo soluções Blue Coat.

Para acessar e acionar as apps Windows ou UNIX:

  1. Clique em Apps no canto superior direito do Splunk Web.
  2. Selecione Launcher na lista do menú drop-down. A App Launcher é apresentada.
  3. Escolha a app Windows ou *NIX (Se estiver usando Windows, você só verá a app Windows; se estiver em uma plataforma UNIX, você só verá a app *NIX) e clique em Launch App.
  4. Se a app ainda não estiver ativa, clique no link para ativar a app.
  5. Siga estes passos para configurar a app de forma a se adequar ao seu sistema.

Outros métodos de Entrada

Existem outras formas de introduzir dados no Splunk. Aqui temos algumas opções populares:

Coletar informações de diversos equipamentos em um ambiente distribuído

Você está imaginando como fazer para trazer ao Splunk informações de um ambiente distribuído, como servidores de aplicação ou web que geram logs locais? Splunk pode ser configurado como um lightweight forwarder (encaminhador peso-leve) e implantado em dúzias ou mesmo centenas de milhares de servidores para capturar dados em tempo real e enviá-los a um indexador Splunk central. Use ${_("Splunk forwarders")} para enviar informações ao Splunk a partir de outros sistemas. Configure Forwarding a partir do Splunk Manager.

Gere scripts para sua coleta de dados

Crie um ${_("scripted input")} para suas fontes de dados customizadas. Scripted inputs são úteis para ferramentas de linha de comando como vmstat, iostat, netstat, top, etc. Obtenha dados de APIs e outras interfaces de dados remotas e filas de mensagens (message queues) e crie métricas e informações de status a partir da execução de comandos de estado de sistema e aplicação tais como vmstat, iostat, etc. Várias apps disponíveis na SplunkBase possuem scripts de entrada para aplicaÇões específicas. Configure scripted inputs a partir do Splunk Manager.

Monitore mudanças de file system

Interessado em saber que mudanças estão ocorrendo em seu sistema de arquivos?(file system)? configure ${_("file system change monitoring")} e veja cada mudança conforme elas ocorrem. Use este método para monitorar arquivos críticos, arquivos de configuração e etc para finalidades de garantia de compliance bem como para encontrar mudanças com impacto em sistemas e acessos não autorizados por segurança e operações.

Interação com os resultados

Os resultados de suas buscas são tão interativos quanto o período de tempo. Nesta sessão você verá como, com apenas um clique, você pode acrescentar, remover e excluir termos de suas buscas.

  1. Navegue até a app Search.
    2. Me leve até lá...
  2. Realize uma busca por qualquer termo em seus dados.
  3. Movimente seu mouse por sobre os resultados da busca. Perceba que palavras e frases são destacadas conforme você movimenta o cursor por sobre elas, indicando que você pode acrescentar estes termos à sua busca.
  4. Destaque e clique em um termo em seus resultados de busca. Sua busca é atualizada para incluir este termo na barra de busca e os resultados que não a contém são filtrados.
  5. De forma alternativa, clique em um termo em seu resultado de busca que já esteja destacado ; Splunk removerá este termo de sua busca.
  6. Além disto, você pode especificar termos que deseja que o Splunk exclua. Destaque um termo e clique sobre ele enquanto mantém a tecla ALT pressionada (no Windows, tecla CTRL); Splunk atualiza sua busca para excluir este termo com uma operação Booleana NOT.

Utilize campos em suas buscas

A busca livre é simples e poderosa, mas nem sempre traz a resposta que você quer. Por exemplo, você pode querer excluir eventos que contém o HTTP status code 200. Mas, se você simplesmente realizar uma busca por "NOT 200", você também irá remover eventos que poderia querer, como por exemplo eventos "503" vindos de um endereço IP que contém 200.

Conforme Splunk indexa cada ítem em suas informações originais, ele descobre e acrescenta campos baseado em pares de nome/valor , cabeçalhos (headers) ou outras informações que são evidentes. Agregar conhecimento

Splunk leva a busca até onde ela nunca esteve ao extrair automaticamente conhecimento de dentro de sua informação de TI e permitindo que você acrescente seu próprio conhecimento em tempo real. Acrescente conhecimento a respeito de eventos, campos, transações, padrões e estatísticas em suas informações. Você pode também identificar, nomear e rotular esta informação.

Splunk mapeia todo este conhecimento durante a busca, assim você pode acrescentar novos campos e tipos de eventos sempre que necessário, sem a necessidade de re-indexar os dados. Desta forma, depois de encontrar todos os eventos de um determinado nome de usuário, você pode imediatamente gerar estatísticas a respeito de atividades específicas desse usuário.

Classificar eventos similares

Quando realiza buscas em sua informação, você está basicamente se livrando de todos os dados não desejados; os resultados de sua busca são eventos que compartilham de alguma característica em comum, e você pode dar a eles um nome coletivo ou "event type"(tipo de evento). Os nomes de seus tipos de eventos são acrescentados como valores dentro de um campo eventtype. Rotular valores de campos

Tags (rótulos) ajudam a agrupar resultados de buscas que compartilham valores de campo. Uma tag é o nome que você dá a um valor ou campo em particular, tais como eventtype, host, source ou sourcetype. Por exemplo, você pode rotular valores de dispositivos com um nome de serviço ou uma anotação indicando que estão de acordo com uma regulamentação como PCI.

Em geral, você pode usar tags para:

  • Ajudar a rastrear valores abstratos de campos, tais como endereços IP ou números de ID que você pode agrupar por localidade ou nome.
  • Utilize uma tag para agrupar um conjunto de valores de campo, para que você possa realizar buscas com um único e simples comando.
  • Utilize tags múltiplas para campos extraídos específicos refletindo diferentes aspectos de sua identidads, permitindo que você realize buscas baseadas em tags para rapidamente filtrar os resultados de que realmente precisa.

Existem duas maneiras de realizar buscas por uma tag: você pode buscar pela tag através de todos os eventos ou em um campo em particular.

Use o Report Builder

Execute o report builder para criar e formatar seu relatórios.

  1. Navegue até a app Search.
    2. Me leve até lá...
  2. Execute uma busca por qualquer termo ou campo que você deseja reportar.
  3. Na barra de status sobre a linha de tempo, você verá um link para Build report link. Clique neste link para lançar o Report Builder em uma janela separada. (Você não precisa esperar pelo fim da busca para poder ativar o Report Builder.)
  4. O Report Builder abre na página Define Report Contents (Definir Conteúdo do Relatório). Use esta página para definir os parâmetros de seu relatório a partir de uma série de listas dropdown. Comece selecionando um Report type (Tipo de relatório). Perceba que o tipo de relatório que você escolhe afeta as demais opções à sua disposição para definição de conteúdo de relatório.
  5. Clique Next Step (Próximo Passo): Format Report (Formatar Relatório) quando tiver terminado de definir o conteúdo de seu relatório. Splunk te levará à página Format report, que apresenta um rascunho do relatório baseado nas configurações de conteúdo que você selecionou.
  6. Clique Format report para mudar a formatação do relatório. Por exemplo, se Splunk te apresentou um relatório em colunas e você acha que um gráfico em linhas teria uma melhor aparência, realize a troca. Existem diversas outras opções de formatação disponíveis. Crie um nome para o seu gráfico, modifique o posicionamento da legenda e mais.
  7. Pronto! Agora você pode salvar o relatório, exportar seus resultados para um arquivo, imprimir ou copiar a URL dos resultados que você pode então compartilhar com os outros usuários Splunk em sua empresa.

Pré-visualização de relatórios

Quando você gera um relatório, Splunk pode apresentar uma prévia dos resultados do relatório enquanto a busca é executada. Esta funcionalidade poupa tempo, em especial quando executando buscas através de longos períodos de tempo. Note que esta prévia está habilitada por padrão para buscas que utilizam comandos de relatório, portanto experimente em um relatório que cobre um grande período de tempo.

  1. Navegue até a app Search.
    2. Me leve até lá...
  2. Insira um relatório na barra de busca.
  3. Enquanto o relatório está sendo executado, a prévia dos resultados será apresentada.
  4. Se você não estiver satisfeito com o relatório e desejar modificá-lo antes do final da execução, clique no botão cancel e edite os termos de busca.

Construir dashboards (painéis de controle)

Incorpore buscas e relatórios na UI (Interface do Usuário) criando um painel de controle. Isto disponibiliza os seus relatórios mais úteis e informativos para outros usuários. Tudo o que você precisa fazer é criar e salvar um conjunto de relatórios que você deseja apresentar na forma de um painel de controle.

  1. Navegue até a app Search.
    2. Me leve até lá...
  2. Selecione o dropdown Actions (Ações) e escolha Create new dashboard (Criar novo painel de controle).
  3. A janela Create new dashboard aparece. Nomeie o painel com dois nomes: Um curto (Dashboard ID) e outro longo (Dashboard name). Clique em Create (Criar) quando estiver pronto.
  4. Inicialmente, seu painel está vazio. Clique Edit the dashboard para abrir a janela Edit e selecione o tipo de painel e busca salva para o seu primeiro painel. Clique em Add panel para acrescentar seu novo painel ao painel de controle.
  5. Clique Edit panel se desejar renomear o painel, modificar seu formato ou atualizar a busca na qual ele está baseado.
  6. Repita os dois últimos passos para criar mais painéis. Arraste-os até encontrar a distribuição desejada.
  7. Você terminou! Clique Close (Fechar) para ver a aparência do seu painel. Se vir algo que deseja mudar, vá até o menú Action no topo da página, selecione Edit dashboard... e arrume ele.
