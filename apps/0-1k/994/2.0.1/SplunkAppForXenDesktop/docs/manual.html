Introduction

About the Splunk App for Citrix XenDesktop

The Splunk App for Citrix XenDesktop consumes desktop delivery session information, Active Directory logs, Windows desktop perfmon metrics (including ICA metrics) and logs.

If you're running a Citrix XenDesktop deployment with thousands to tens of thousands of virtual desktops, Splunk for XenDesktop can make your job easier. The Splunk App for XenDesktop provides deep visibility into every component of your XenDesktop infrastructure helping you:

  • Diagnose and troubleshoot user issues
  • Proactively monitor service levels and operational metrics in real time
  • Report on session activity, volumes, load times, user experience and adoption
  • Perform real time as well as historical analysis to prevent issues from impacting end users
  • Monitor configurations of desktop images and changes made to them
  • Monitor application usage and configurations inside desktop VMs, allowing operational capacity planning
  • Monitor desktop application usage and plan desktop sizing

Splunk can also correlate data from a wide variety of sources such as gateways, network devices, storage devices, WAN accelerators, and load balancers by time, allowing you to see the full context of events and problems.

How does it work?

Splunk forwarders pull logs and performance metrics from the following components of your XenDesktop deployment:

  • Desktop Delivery Controller servers (referred to as DDCs)
  • Desktop virtual machines (referred to as VDAs)
  • Citrix Licensing Server (Splunk App for Citrix XenDesktop version 2.0 and later only)

and forwards them to a central Splunk instance running a custom-built for XenDesktop UI that provides prebuilt alerts, reports and dashboards.

Before you deploy

Platform and hardware requirements

This topic discusses the underlying requirements for running the Splunk for Citrix XenDesktop app in production.

What versions of Splunk are supported?

All instances of Splunk, including the forwarders, should be version 4.3 or later. Be sure to download the correct version for your platform. Running a 32-bit version of Splunk on a 64-bit platform will negatively impact performance.

What versions of XenDesktop are supported?

XenDesktop 4, 5, 5.5, and 5.6 are supported.

Hardware and OS requirements

A Splunk for XenDesktop deployment includes multiple components (described in more detail in What a Splunk for XenDesktop deployment looks like). These components can run on different platforms, so be sure you're downloading the correct platform and architecture (32- or 64-bit).

  • For details about supported OSes, refer to "System requirements" in the core Splunk product documentation.

The "central Splunk instance"

The Splunk instance running the Splunk for XenDesktop app and indexing the data (what we'll call the "central Splunk instance" for this deployment) can be any OS currently supported by Splunk.

The level of hardware required for your central Splunk instance is highly dependent on the volume of data coming from the DDC and the VDAs.

  • For details on adjusting the volume of data coming from your DDC and VDAs, refer to "Other deployment considerations" in this manual.
  • For guidance on scaling your deployment, check out "Capacity planning" in the core Splunk product documentation.

The other XenDesktop components

The systems running the other components of the Splunk for XenDesktop deployment (the Licensing Server, DDC and the VDAs) are Windows systems. Make sure you download the correct platform and architecture (32- or 64-bit) for your hardware. They are not doing any indexing, so the level of hardware for these components is less critical than for the central Splunk instance.

PowerShell v2.0 is required on the Citrix Licensing Server, DDC and VDAs

What data are collected

The Splunk App for Citrix XenDesktop and its associated add-ons collect data from your XenDesktop components and index it so that it can be used to generate the dashboards and reports shown in the Splunk App for XenDesktop GUI. This topic discusses the specifics of the data being collected.

Data collected from the DDC

The TA-[XD4|XD5]-Broker add-on (use XD4 or XD5 for XenDesktop version 4 or 5 respectively) collects information about the number of sessions and desktops served by the DDC.

Data collected from the VDAs

The TA-XD-VDA add-on collects the following data:

  • Windows WMI data (OS info and patch level)
  • Windows Perfmon (cpu, memory, disk space, network interface, running processes)
  • Also gathered via Perfmon is the Citrix-specific ICA data about the performance of and activity on each VDA, such as network usage, performance of sound, mouse clicks, and so on. For complete list of VDA counters that the Splunk for XenDesktop app collects, refer to http://support.citrix.com/proddocs/topic/infocenter/ic-how-to-use.html on the Citrix support site.

Data collected from the Citrix Licensing Server

The TA-CitrixLicensing-1 add-on collects information about licenses in total, in use, and percentage by product via Powershell and WMI.

Other deployment considerations

This topic covers important considerations you should review and understand before proceeding with a production Splunk deployment.

Storage considerations

As with all Splunk deployments, it's important to ensure you have enough disk space to accommodate the volume of data your indexers will be processing.

  • For a general discussion of Splunk data volume and how to estimate your storage needs, refer to "Estimate your storage requirements" in the core Splunk product documentation.

What volume of data should I expect from my XenDesktop deployment?

For details on the types of data that Splunk collects from your XenDesktop deployment, refer to "What data are collected".

Each VDA can produce on the order of 7-10MB of data per day, the great majority of which is the ICA data metioned in "What data are collected". This means you will have 7-10MB times the number of VDAs in data arriving at your Splunk indexer, plus the data from the DDC, which is about 500k per session, per desktop served.

This volume is tunable to meet your requirements once you've deployed Splunk as described later in this manual.

To tune the VDA data levels, you can change the interval at which Perfmon data is sampled by editing a copy of perfmon.conf that the deployment server pushes out to the VDAs as part of the Xendt_agent add-on.

1. Copy the version of perfmon.conf that's in $SPLUNK_HOME\etc\deployment-apps\xendt_agent\default and put it in $SPLUNK_HOME\etc\deployment-apps\xendt_agent\local.

2. Change the value of the interval field for each counter to suit your needs.

3. Using the CLI, from $SPLUNK_HOME\bin on the instance of Splunk running on the DDC, type splunk reload deploy-server. This will tell the deployment server to push out the updated configuration to the VDAs, which will then change the interval at which they forward their data to the indexer.

Note: Be sure not to edit the copy in the default directory, or it will be overwritten when you upgrade.

Performance considerations

Your performance and performance requirements will depend on the scale of your Splunk deployment. If you are expecting to scale your Splunk deployment beyond 1-2GB per day of indexed data, and expect to have more than 2 users searching this data (or using the Splunk App for XenDesktop) concurrently, consider distributing your deployment.

  • For details on capacity planning, refer to "Hardware capacity planning" in the core Splunk product documentation.

Licensing information

As with any Splunk deployment, your licensing requirements are driven by the volume of data your indexer processes. Use the information in the "Storage considerations" section above to determine your licensing volume. Contact your Splunk sales representative to purchase additional license volume.

  • Refer to "How licensing works" in the core Splunk product documentation for more information about Splunk licensing.

Permissions

All components of your Splunk App for XenDesktop deployment can safely run as the Local System User.

  • For additional details about the permissions Splunk needs to run on Windows, refer to "Choosing the user Splunk should run as" in the core Splunk product documentation.

Backups and archiving

You can configure Splunk to back up both your indexed data and configuration data. Splunk can be configured to delete data based on either the size of indexes or the age of data in indexes. By default, Splunk will delete data if all the data in a given archived index are more than 6 years old.

  • For details on configuring backups in Splunk, refer to "What you can back up" and other topics in the same chapter in the core Splunk product documentation.
  • For details on configuring backups in Splunk, refer to "Back up indexed data" and other topics in the same chapter in the core Splunk product documentation.
  • For details on configuring archive policy in Splunk, refer to "Set a retirement and archiving policy" in the core Splunk product documentation.

Deploy the Splunk App for XenDesktop

What a Splunk App for XenDesktop deployment looks like

A Splunk App for XenDesktop deployment is made up of the following components:

  • The "central" Splunk instance which includes the indexer and Splunk Web interface, to which you will add the Splunk App for XenDesktop. The app contains the UI components, searches, and indexing definitions for your XenDesktop data. This system will receive the data from the other components.
    • In larger Splunk environments, this central instance can be distributed such that the Splunk Web interface is served from one or more distributed search heads and searches across multiple indexers, known as search peers. For more information about installing your central Splunk instance in a distributed fashion for scalability, review the "Distributed Deployment Manual" in the core Splunk product documentation set.
  • The Desktop Delivery Controller (DDC), onto which you will install the Splunk universal forwarder, and then install:
    • the Splunk TA-[XD4|XD5]-Broker add-on (XD4 or 5 depending on what version of XenDesktop you are running) and
    • the TA-ForwardServer add-on. This system will forward DDC session data to the indexer.
  • Some number of Virtual Desktop Agents (VDAs), onto which you will install the Splunk universal forwarder and then:
    • the TA-XD-VDA add-on and
    • the TA-ForwardServer add-on. The VDAs will also forward their data to the Splunk indexer.
  • Optionally, you can also install a Splunk universal forwarder onto your Citrix Licensing Server and install
    • the TA-CitrixLicensing-1 add-on and
    • the TA-ForwardServer add-on.

If you have an existing Splunk deployment that includes a Splunk deployment server, you can also use it to deploy the various add-ons to the VDAs, DDC, and the Citrix Licensing Server as described in "Install the add-ons onto the VDAs, DDC, and Citrix Licensing Server"

Additional components you can Splunk

You can also Splunk other components of your infrastructure for additional correlation:

  • LDAP
  • Network devices
  • Gateways
  • Hypervisors
  • Storage arrays
  • and anything else in your infrastructure that logs data.

Install the XenDesktop app on the Splunk indexer

The central component of a Splunk for XenDesktop deployment is the Splunk indexer. This component includes a full standard install of Splunk, which includes Splunk Web, to which you will add the Splunk for XenDesktop app.

As discussed in "Platform and hardware requirements" and "Other deployment considerations", this machine must be provisioned to support the level of indexing and interaction with the Splunk Web interface you anticipate for your deployment.

This Splunk instance will be acting as a receiver. It will receive data from the DDC, the VDAs, and optionally, the Citrix Licensing Server, which will in turn be configured as forwarders.

Step 1: install Splunk

Download the full Splunk package for your platform, and follow the installation instructions in the core Splunk documentation.

You can use an existing Splunk instance, as long as it meets the aforementioned performance requirements and isn't too busy making other peoples' lives easier.

Step 2: install the Splunk for XenDesktop app

This procedure assumes you have already installed Splunk on the host you intend to use as the indexer for your XenDesktop data.

Download the Splunk for XenDesktop app from Splunkbase.

1. Copy the app file (SplunkAppForXenDesktop.tar.gz) to a location that is accessible from the host on which you installed Splunk.

2. Log in to Splunk Web and navigate to Manager > Apps.

3. Click Install app from file and follow the instructions to locate and upload the file mentioned in step 1, and then restart Splunk as instructed.

4. Log back in to Splunk.

Proceed to the instructions in the following chapters to continue the installation.

Receiving data

By default, the XenDesktop app configures your instance of Splunk to receive data over port 9997. If you need this to be a different port, you can change this value, and you'll also need to change it in the outputs.conf files on the instances configured to forward to this one.

  • Read "Set up forwarding and receiving"in the core Splunk product documentation for an introduction to forwarding and receiving.
  • Read "Configure forwarders with outputs.conf" in the core Splunk documentation for information about outputs.conf.

Install the universal forwarder on the VDAs, DDC, and Citrix Licensing Server

This topic discusses installing the Splunk universal forwarder onto all the XenDesktop systems from which you wish to collect data. The universal forwarder serves as the basic Splunk platform upon which you will then install specific add-ons containing configuration files that define what data to collect and where to send it.

  • If you you have a large XenDesktop deployment, you probably want to use your existing automated software distribution methodology to distribute the correct universal forwarder packages out to their respective XenDesktop role hosts, and then use the Splunk deployment server to deploy the add-ons.

Distribute the universal forwarder packages to XenDesktop hosts using automation

This procedure describes how to set up the universal forwarder to be installed using your environment's pre-existing software distribution methodology, and assumes you'll be using the Splunk deployment server to deploy the necessary add-ons once this is complete.

Note: If you are deploying the add-ons by hand instead of using the Splunk deployment server, read "Install the add-ons manually" for details on what to put where.

Customize the universal forwarder packages before distributing them

If you are going to use the Splunk deployment server to deploy and maintain the add-ons described in "What a Splunk for XenDesktop deployment looks like", you must first edit the universal forwarder package to inform that system what role it is playing in the deployment so that it receives the correct add-ons from the deployment server. For example, you'll want to configure the universal forwarder package being installed on all the VDAs to accept the TA-XD-VDA add-on, whereas the DDC's universal forwarder package should be configured to look for the TA-XD5-Broker (or TA-XD4-Broker if you're running XenDesktop 4) add-on. The result will be a custom universal forwarder package for each XenDesktop role.

To do this, create a copy of the universal forwarder package for each XenDesktop role listed below, and then add a copy of deploymentclient.conf in $SPLUNK_HOME\etc\system\local that contains the information shown before pushing it out to to the appropriate host(s), being sure to substitute the hostname or IP for your deployment server as shown:

XD4-Broker

[deployment-client]
clientName = xd40broker

[target-broker:deploymentServer]
targetUri = <hostname or IP for deployment server>:8089
XD5-Broker

[deployment-client]
clientName = xd50broker

[target-broker:deploymentServer]
targetUri = <hostname or IP for deployment server>:8089
Citrix Licensing Server

[deployment-client]
clientName = ctxlic

[target-broker:deploymentServer]
targetUri = <hostname or IP for deployment server>:8089
VDA

[deployment-client]
clientName = xdvda

[target-broker:deploymentServer]
targetUri = <hostname or IP for deployment server>:8089

Note: The TA-Deployment add-on also includes examples of deploymentclient.conf configurations for deploying the various add-ons to their respective forwarders. Use the information in the example configuration files to create a copy of deploymentclient.conf in each add-on's $SPLUNK_HOME/etc/system/local/ directory to set the value of targetUri to the deployment server.

Using a shared/distributed image

If you are using a shared image, you must include a startup script that sets the hostname of the client or all the data coming into the Splunk App for XenDesktop dashboards will be marked as coming from your master image host. The following is an example of such a script:


@echo off
"C:\Program Files\splunkuniversalforwarder\bin\splunk.exe" cmd btool.exe --user=nobody --app=system server delete general guid
"C:\Program Files\splunkuniversalforwarder\bin\splunk.exe" cmd splunkd rest POST /services/server/settings/settings host=%COMPUTERNAME%
"C:\Program Files\splunkuniversalforwarder\bin\splunk.exe" cmd splunkd rest POST /services/server/settings/settings serverName=%COMPUTERNAME%
"C:\Program Files\splunkuniversalforwarder\bin\splunk.exe" start

Note: If you are using a PVS image or other distributed image, make sure you set the startup of Splunk services to manual.

Distribute the universal forwarder packages

1. Download the standalone universal forwarder package (not the full Splunk package) from the universal forwarder download page on splunk.com. Ensure that you download the correct architecture and OS packages for your environment.

2. Use your existing software distribution methodology to deploy the universal forwarder package to each VDA, the DDC, and to the Citrix Licensing Server, using the following command in your script or other installation tool to complete the installation:

msiexec.exe /i splunkuniversalforwarder-[version]-x[32|86]-release.msi DEPLOYMENT_SERVER="IP ADDRESS OF DEPLOYMENT SERVER:8089" AGREETOLICENSE=Yes /quiet

Be sure to:

  • specify the correct IP address of the system serving as your deployment server as "IP ADDRESS OF DEPLOYMENT SERVER:"
  • use the correct architecture (32bit or 64bit) version of the package.
  • use the exact name of the msi file for the version of the universal forwarder you are using.

This will auto-start each universal forwarder (but not restart the host machine).

3. When the universal forwarders have all restarted:

  • If you have deployment server configured as described in "Configure the deployment server", log into the Splunk deployment server instance and use the CLI to tell the deployment server to push out the add-ons out to the VDAs, DDC, and Citrix Licensing Server. To do this, navigate to $SPLUNK_HOME\bin and type splunk reload deploy-server . It will then begin pushing out the appropriate add-ons to each VDA, the DDC, and the Licensing Server. Once an add-on is deployed, it will begin forwarding data to the central Splunk instance (with the indexer).
  • If you are deploying the add-ons by hand, read "Install the add-ons manually" for details on what to put where.

Install the add-ons onto the VDAs, DDC, and Citrix Licensing Server

Once you've installed the Splunk App for XenDesktop and installed the universal forwarder on the VDAs, DDC, and Citrix Licensing Server, the next step is to configure the add-ons for your environment and then deploy them to the appropriate universal forwarders. These add-ons are unpacked with the app into their own $SPLUNK_HOME/etc/apps/SplunkAppForXenDesktop/appserver/addons/ directories:

  • The VDA add-on, TA-XD-VDA goes on all VDAs and configures inputs to collect data to send to the Splunk indexer.
  • The DDC/Broker add-on (TA-XD4-Broker or TA-XD5-Broker depending on what version of XenDesktop you're running) goes on the DDC. This add-on configures inputs to collect data to send to the Splunk indexer.
  • The Citrix Licensing Server add-on (TA-CitrixLicensing-1) configures inputs to collect data to send to the Splunk indexer.
  • The forwarder add-on (TA-ForwardServer) goes on all the universal forwarders. You will customize this add-on to send data to the indexer.
  • The deployment server add-on (TA-Deployment) can be installed on any full Splunk instance (including your central Splunk instance) to configure it to deploy the rest of the add-ons listed above to their respective universal forwarders. This add-on enables the deployment server functionality to deploy the correct add-ons to the other components (the VDAs, DDC, and Citrix Licensing Server). If you are already running deployment server on a Splunk instance in your environment, use the configurations within the add-on package to add to your existing copy of serverclass.conf.

Edit the TA-ForwardServer add-on to point to the indexer(s)

Before you deploy the TA-ForwardServer add-on to all the components in this deployment, you must edit a copy of outputs.conf in $SPLUNK_HOME/etc/deployment-apps/TA-ForwardServer/local/ to specify your indexer or group of indexers as the target group. For more information on outputs.conf, forwarding and target groups, refer to "Configure forwarding with outputs.conf" in the core Splunk product documentation.

Configure deployment server to deploy the add-ons

If you have an existing Splunk environment that includes a deployment server, you can use it to deploy the add-ons to the other systems. The TA-Deployment add-on included in the Splunk App for XenDesktop contains the necessary configurations in its own /default/app.conf and /default/serverclass.conf files to deploy the correct add-ons to the other components (the VDAs, DDC, and Citrix Licensing Server).

For information on how deployment server works, refer to "Deploy apps and configurations" in the core Splunk documentation.

1. From your central Splunk App for XenDesktop instance, navigate to the $SPLUNK_HOME/appserver/addons/ directory.

2. Copy the contents of the TA-Deployment directory to $SPLUNK_HOME/etc/apps/TA-Deployment/ on the Splunk instance that is acting as your deployment server. This installs the TA-Deployment add-on but does not enable it.

  • If you are already running deployment server on a Splunk instance in your environment, use the /default/serverclass.conf configurations within the add-on directory to add to your existing copy of serverclass.conf.

3. Copy the contents of the TA-CitrixLicensing-1 directory to $SPLUNK_HOME/etc/deployment-apps/TA-CitrixLicensing-1/.

4. Copy the contents of the TA-XD40-Broker or TA-XD5-Broker directory to $SPLUNK_HOME/etc/deployment-apps/TA-XD4-Broker/ or $SPLUNK_HOME/etc/deployment-apps/TA-XD5-Broker/ (depending on whether you're running XenDesktop 4 or 5 on your DDC).

5. Copy the contents of the TA-XD-VDA directory to $SPLUNK_HOME/etc/deployment-apps/TA-XD-VDA/

6. Copy the contents of the TA-ForwardServer to $SPLUNK_HOME/etc/deployment-apps/TA-ForwardServer/

7. Ensure that the copy of outputs.conf in $SPLUNK_HOME/etc/deployment-apps/TA-ForwardServer/default/ has been edited to specify your indexer or group of indexers as the target group.

Perform the next step after you have completed the steps in "Install the universal forwarder on the VDAs, DDC, and Licensing Server":

8. Navigate to $SPLUNK_HOME/bin and type splunk restart

This notifies the VDA, DDC, and Citrix Licensing Server hosts that they need to get their respective add-ons. Once the add-ons are pushed out to each VDA, DDC, and Licensing Server host, the Splunk forwarder process will restart and then begin forwarding data to the central Splunk instance.

Install an add-on manually

If you're setting up a test environment, or have a small XenDesktop environment, you can install all the add-ons individually by hand. To do this:

1. Install the universal forwarder on the VDA, DDS, or Citrix Licensing Server by following the instructions in "Install the universal forwarder on Windows from the commandline"."

2. Copy the add-on directory to $SPLUNK_HOME\etc\apps\<add-on_name> on the appropriate VDA, DDC, or Licensing Server host. Make sure you copy both the role-specific (VDA, DDC/Broker, or Licensing Server) add-on and then also the forwarder add-on. Make sure to edit the outputs.conf in $SPLUNK_HOME/etc/apps/TA-forwardserver/local to point to the indexer.

3. Restart Splunk on that system. Navigate to $SPLUNK_HOME/bin and type splunk restart

Use the Splunk App for XenDesktop

Log in and get started

This topic shows you how to log in to Splunk Web, access the Splunk for XenDesktop app, and get started.

Log in to Splunk Web

To log into Splunk Web and access the Splunk for XenDesktop app, navigate to:

http://<host>:8000

Use the host and port you chose during installation. The default port is 8000.

The first time you log in to Splunk, the default login details are:
Username - admin
 Password - changeme

Splunk recommends that you change the admin password to a secure password.

Access the Splunk for XenDesktop app

Once you've logged in to Splunk Web, you'll see Splunk Home, which lists all the apps that are currently installed. You should see the Search and Getting Started apps, as well as the Splunk for Citrix XenDesktop app.

To access the Splunk for Citrix XenDesktop app, click on it in the list. You'll see the "Environment" page. If it has been fewer than about 45 minutes since the VDAs, DDC, and Citrix Licensing Server started forwarding data to the central Splunk instance, the tables and charts in the app UI may not have populated fully yet. To kick off this process manually, navigate to Reports > Virtual Desktop Lookup Table > Rebuild Desktop Lookup Table and click Submit. Once this process has completed, all the rest of the reports and views in the app will have data.

User Reports

  • User List: This report provides details about users who have logged into your environment. Adjust the time range for this report by changing the timerange picker. Filter this view by entering values in the fields below and pressing Enter. Click on a panel to drill into the session details for that element.

Desktop Reports

  • Virtual Machine List: This report provides details about the virtual desktops in your environment. Adjust the time range for this report by changing the timerange picker. Filter this view by entering values in the fields below and pressing Enter. Click on a panel to drill into the session details for that element.
  • Desktop Inspector: This report profiles the performance counters and other chareteristics of a virtual desktop. Enter the hostname of the virtual Desktop, select a timerange and press Submit. Select a row for a detailed report for that counter. To view the entire result set or for this report or manipulate the data further, click View Full Report.
  • Process Inspector: This report inspects a running process on a virtual desktop. Enter the name of the process and hostname of the virtual desktop and press Submit. Span refers to the size of the time buckets for the selected performance counter and desktop. For example span=1h with a time range of Last 24 hours will bucket the results into 24, one hour time buckets. Toggle between a graphical Timechart view and a Results Table using the links in each panel. To view the entire result set for a given counter or manipulate the data further, click View Full Report.
  • Desktop Variance: This report compares a desktop's behavior over time with the typical utilization for that host in the selected time range by establishing a baseline for comparison. Enter the hostname of the virtual desktop, select a timerange and press Submit.
  • Desktop Outliers: This report compares a selected performance counter for a group of hosts and determines where the outliers are. The average counter value for each host is compared to a baseline average for the entire group. Enter the hoststring of the virtual desktops you would like to compare, select a PerfMon counter, timerange and press Submit.

ICA Reports

  • ICA Statistics: This report provides session bandwidth and latency information for your virtual desktop sessions. Sessions in the High Latency or Low Bandwidth table are sessions that surpass a defined threshold. Filter this view by entering values in the fields below and pressing Enter. For detailed views on other counters, go back to the Reports menu and choose Environment. To view the entire result set or for this report or manipulate the data further, click View Full Report.
  • ICA Session Inspector: This report compares the ICA Session Bandwidth and Latency to a selected WindowsPerfMon counter. Enter the hostname of the virtual desktop and a user, select a timerange and press Submit. Span refers to the size of the time buckets for the selected performance counter and desktop. For example span=1h with a time range of Last 24 hours will bucket the results into 24 one-hour time buckets.

Performance Reports

These dashboards show details for memory, cpu, network, and disk usage.

Alert Analyzer

This report compares the user experience alerts and scores for a set of desktops and/or users. Enter a hoststring and userstring for the groups you would like to compare and press Submit. Click into the report to see Alert details for the selected entity. Filter this view by entering values in the fields below and pressing Enter. To view the entire result set for this report or manipulate the data further, click View Full Report.

Citrix Licensing

This report provides details about the Citrix Licenses in your environment.

VDI Error Console

This dashboard displays Windows Event Log errors and warnings from the Winevent logs on your virtual desktops. Use the timerange picker to adjust the time range for the dashboards. Filter this view by entering values in the fields below and pressing Enter. To view the entire result set for this report or manipulate the data further, click View Full Report.

Virtual Desktop Lookup Table

  • View Desktop Lookup Table: This view displays the information stored in the XenDesktop lookup table for your deployment. Filter this view by entering values in the fields below and pressing Enter.
  • Rebuild Desktop Lookup Table: This view rebuilds your XenDesktop lookup table, overwriting the existing file, using the sourcetype=vdi:xendesktop:*:desktop sourcetype. This view is typicallyonly used on first time run.
