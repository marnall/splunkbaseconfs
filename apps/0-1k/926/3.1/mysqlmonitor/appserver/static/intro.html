Introduction

This is the application for MySQL developed by Remora Technologies. The app is for viewing and investigating data produced primarily from MySQL's global status variables, which are collected on a schedule. Saved searches and dashboards exist for the most common points of interest in a production MySQL instance.

Version 3.0 has been written to use the offical Splunk DB Connect application.

Please refer to the legacy page regarding the earlier versions of this application.

The application consists of:

  • Database inputs with defined SQL queries to gather data from MySQL
  • Sourcetypes to extract useful values from the incoming data
  • Dashboards and macros making investigation of this data easy
The primary database input used by this application is the database status variables and slave status inputs, additional inputs have been provided to gather more data than is currently used by any of the dashboards, so ad-hoc searches can be used to investigate less common metrics.

This application has been tested on:

  • MySQL 5.1
  • Percona Server 5.1
  • Percona Server 5.5
There is no reason the application should not work with many other recent versions of MySQL.

Splunk configuration overview

To configure your Splunk server you should:

  • Install and configure the Splunk DB Connect application
  • Define your database connection(s) in the Splunk DB Connect application
  • Ensure adequate permissions for your MySQL connections on the MySQL server
  • Configure application database inputs
  • Configure server groups for comparison graphs

This version creates an index named "mysql" defined in $SPLUNK_HOME/etc/apps/mysqlmonitor/default/indexes.conf.

The database inputs can either be configured via the Manager app or the local inputs.conf file. The sourcetype for each of the inputs should be as shown in the sample inputs.conf file that is provided.

The application allows you to configure server groups, which allows the overlaying of multiple servers on a single graph for comparison. To do this, each group must be discernable via a glob expression on the hostname that matches all servers in that group and no others -- for example "myappdb*" would match "myappdb1" and "myappdb2" allowing comparison of these two hosts in the dashboards. The groups can be set up in $SPLUNK_HOME/etc/apps/mysqlmonitor/lookups/dbgroups.csv A sample file is provided showing how to define a group.

Database inputs

The pre-defined database inputs are to be configured as standard database data inputs on a Splunk indexer or forwarder. Each database data input is resposible for a different type of metric gathering, which may or may not be required for a given host:

  • statusvars - Most metrics are delivered by this input, from the MySQL global variables and status
  • tablestats - Metrics on the tables in your schema
  • slavestatus - Include this if the monitored server is a replication slave
  • processlist - Reads currently running processes

The "statusvars" and "slavestatus" are used when generating the included dashboards, but the others provide useful metrics for ad-hoc queries or your own dashboards.

Quick start

This is a quick start guide for setting up the Remora App for MySQL in your MySQL environment.

Create and define MySQL connections

Example SQL for creating a new MySQL user account for monitoring your MySQL instance in Splunk:

  CREATE USER splunk@'%' IDENTIFIED BY '****';
  GRANT SELECT, PROCESS, REPLICATION CLIENT ON *.* to splunk@'%';
  FLUSH PRIVILEGES;

Define your MySQL connections in the Splunk DB Connect application.

  1. Navigate to Apps > Splunk DB Connect > Manage database connections.
  2. Select the "New" button on the "External Databases" interface.
  3. Enter a short descriptive name for the "Name" field (e.g. "db01"), make note of this name.
  4. Select "MySQL" from the "Database Type" dropdown menu.
  5. Enter the host, port (optional), database name (this is not important, but required e.g. "information_schema"), username and password.
  6. Select the "Save" button to create the database connection.

Configure application inputs

  1. Create $SPLUNK_HOME/etc/apps/mysqlmonitor/local/inputs.conf file based on the sample file.
  2. Replace the first argument in the input stanzas (e.g. "db01") with the database name you created above.

Restart Splunk!
