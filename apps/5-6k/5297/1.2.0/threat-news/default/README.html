Introduction

As I said in my first “Weekly Intelligence Summary”, I want to provide the latest cyber threat intelligence updates to some of my clients or friends who are working in the cybersecurity roles for the financial industries in Hong Kong. The term Cyber Threat Intelligence (CTI) or Threat Intelligence (TI) may have different meanings for different people. I have no intention provide you my definition in this presentation. I just want to share the joys and pains on creating these summaries and my belief on the statement of “IR is guided by CTI and CTI lives on the information collected from IR”.

 

Each week’s Summary is prepared after quick and dirty review from various subscribed feeds or sources that I found. Those tactical and technical intelligence are manually updated to my MISP instance (https://cisp.org.hk). On each Friday, select feeds are extracted from the Dashboards which are pulling from the MISP instance.

 

The first time I aware the term Cyber Threat Intelligence can be dated back in 2013-14. I gave a presentation with title of “Tracing APT_163QQ”. In that presentation, I had identified the IOCs, such as: blackteae.com, nanmu20088@163.com, 315612477@qq.com, cnxinghang@163.com and a whole block C-class IP addresses in Hong Kong. These IOCs are plot on a data mining map with Maltego.

 

In 2016, I presented my work called Maltelligence (malware-intelligence) in Cyber Security Symposium. The tool I created can parse IOCs from published security blogs or pdf papers and store them in a database. The collected IOCs can be further enriched with linked technical data pulling from OSINT sources. The project is created with intention to allow malware/forensics analysts to perform correlation check on attack campaigns launched against APAC region victims. That is absolutely a coincident, in the same conference, HKMA announcement the iCAST framework which caused the term - Threat Intelligence creeps into the minds of Hong Kong information security professionals.

 

In the past months, some of people asked me to gain access to my Dashboards. I said why don’t you set up one for your internal use.

 

In this SOP, I am going to provide my step by step instructions on how to build and manage the MISP and the dashboards for anyone who want to create and manage their own threat intelligence platform for internal use

STEP BY STEP PROCEDURES:

  1. Install a MISP instance

Install Commands

Depends on how you want to deploy MISP and the Splunk dashboard.

 

I highly recommend using a low-cost cloud platform to setup your MISP. In my case, I use Digital Ocean and using Ubuntu 20.04(LTP) x64 with 1GB/25GB SSD disk, 1000GB therefore you can skip the up2date script. Then, I transfer and my macOS SSH Keys for authentication.

 

Follow the installation instructions provided at MISP Install Documentation to install my version of MISP.

Depends on how you want to secure your ssh connections (in my case, I used SSH key and ufw firewall), create an user account after login as root. In my case, I login as root using your ssh public key and create an user account called “misp”. Put it under sudo group and add your ssh public key to this account’s .ssh directory.

 

% ssh root@dashboard.threatintel.hk

# adduser misp

# usermod -aG sudo misp

# login misp

# mkdir .ssh

# vi .ssh/authorized_keys

$ exit

# exit

% ssh misp@dashboard.threatintel.hk

# sudo apt install net-tools    // intall ifconfig

 

Before you follow the installation procedures described in the Install Document, create your MISP_BASEURL. In my case: “misp2.threatintel.hk”.

 

# sudo wget -O /tmp/INSTALL.sh https://raw.githubusercontent.com/MISP/MISP/2.4/INSTALL/INSTALL.sh

$ bash /tmp/INSTALL.sh -c     // install core && no change of MISP_BASEURL

 

Go to Manual Installation to check the shell scripts to fix the errors. (In my case, I have some warnings and has no error).

 

Finally, the script generated a final message of: “MISP Installed, access here: dashboard.threatintel.hk”. Some files with credential contents were generated. I don’t know whether I need to use such information in the future, therefore, I backup a copy from my ssh terminal and safe on my macOS laptop or at least, you have to download the mysql.txt (which contains the MariaDB misp password for use when configuring the DBconnect to pull the from MISP)

 

Login https://dashboard.threatintel.hk/users/login with Email = admin@admin.test and Password = admin to change my admin password then, configure MISP.

 

First, I go to check and configure settings of “Diagnostics” in “Server Settings & Mainteance” under “Administration”.

 

Some configuration needs further tunings, I refresh the browser after making the following changes. (Please note I have not installed the modules, because at time of writing this SOP, the bash /tmp/INSTALL.sh -A  does not install the module well) 

 

# sudo mysql -u root -p    // no password is require to fix the database schema

MariaDB> use misp;

MariaDB> CREATE INDEX `event_id` ON `event_reports` (`event_id`);

MariaDB> CREATE INDEX `sharing_group_id` ON `event_reports` (`sharing_group_id`);

# sudo apt-get install php7.4-zip

# sudo apt-get install php7.4-intl

# sudo service apache2 restart

 

I have a copy of my old database from previous MISP instance, therefore, I upload it to this new instance and start recovering the database (including login, organizations details and all MISP data).

 

# sudo mysql -u root -p

MariaDB> use misp;

MariaDB> source misp-20201009.sql;     // in case you have restored a previous dump  

 

Now I have a running version of MISP core. I am going to prepare another document to descibe my tricks on how to “maintain” the MISP. The keys are the enrichment, how manually added the attributes and collected IOCs from various sources.

 

  1. IT Infrastructure

Operating Commands

Depends on if you want to use a free version or paid version of Splunk. You may consider to install Splunk either on the same machine or a separate machine. (note: Splunk web interface using port 8000 and MISP web interface using port 80). In this case, I use the same machine to install MISP and Splunk (with DBconnect).

 

If you want to keep your data on the cloud, consider to install and configure firewall between the Splunk machine and the MISP machine.

 

  1. Install Splunk

Operating Commands

Download a free copy of the Splunk Enterprise for Linux (my case: version 8.0.6) after register an account with Splunk.

 

Upload and install the splunk-8.0.6-152fb4b2bb96-linux-2.6-amd64.deb.

 

# apt update

# sudo dpkg -i splunk-8.0.6-152fb4b2bb96-linux-2.6-amd64.deb

# sudo /opt/splunk/bin/splunk start     // provide admin details

 

After the installation, restart Splunk and login again. Next step is download and install java-8-openjdk:

 

# sudo apt install openjdk-8-jre-headless

# sudo /opt/splunk/bin/splunk enable boot-start -systemd-managed 1 -user misp.    // enable splunkd start after booting up

# sudo /opt/splunk/bin/splunk start    

 

Download DBconnect either browse apps from Splunk web or download the app and “install app from file”.  In my case, I first download the app (splunk-db-connect_340.tgz) and use the “install the app from file downloaded” option to install DBconnect. The Splunk needs restart for complete installation of DBconnect.

 

Then configure configure DBconnect by adding the path “/usr/lib/jvm/java-8-openjdk-amd64” to DBconnect  Configuration  Settings  JRE Installation Path. If everything is done correctly, you need to restart Splunk, then login and go DBconnect  Configuration  Settings to see all other parameters be filled up automatically. (Fig. 1)

 

If you have the above configured, download and install Mysql driver, then you have to copy the database connector jar file to the DBconnect directory and refresh the Driver page to see the mysql connector works.

 

# sudo dpkg -i mysql-connector-java_8.0.21-1ubuntu20.04_all.deb

# sudo cp /usr/share/java/mysql-connector-java-8.0.21.jar /opt/splunk/etc/apps/splunk_app_db_connect/drivers

 

You can configure DBconnect’s logging option and check Usage Collection or do it later. Now move to configure Databases.

 

First, Add “New Identities”, (you need the mysql.txt file which contains the MISP database password).

 

Then, Add “New Connection” with configuration as below:

·       Connection Name = misp

·       Identify = misp

·       Connection Type = MySQL

·       Timezone = Asia/Hong_Kong

·       Host = 127.0.0.1 (or IP of the MISP server)

·       Port = 3306 (firewall need to be enabled to access by this IP)

·       Default Database = misp

·       Advance Settings = Read Only

·       Fetch Size: 100


If no error, try accessing the database using “Data Lab“  “SQL Explorer”:

·       Connection = misp

·       Category = misp

·       Table = taxonomies

·       Click Run

 

To use our CTI – Dashboard, we need to pull the data from MISP tables and keep them analyse by Splunk indexes. You can import all of table or just A”dd Input” as per the following:

·       Rising

·       Rising Column: id

·       Checkpoint Vaule: 1 Execution Frequency: 0 * * * * Source Type: mysqld

·       Index: misp

 

attribute_tags

select id, attribute_id, event_id, tag_id from attribute_tags where id > ? order by id asc;

Feath Size: 10000

attributes

select e.date, a.id, a.event_id, a.category, a.type, a.value1, a.value2, a.distribution, a.to_ids, a.object_id, a.object_relation, a.comment, e.id as evt_id from attributes a left join events e on a.event_id = e.id WHERE a.id > ? ORDER BY a.id ASC

Feath Size: 10000

 

event_tags

select id, event_id, tag_id from event_tags where id > ? order by id asc;

Feath Size: 10000

 

events

select date, id, info, threat_level_id from events where id > ? order by id asc;

Feath Size: 1000

 

galaxies

select id, name, type, description, namespace, kill_chain_order from galaxies where id > ? order by id asc;

Feath Size: 300

 

galaxy_clusters

select id, galaxy_id, type, value, description, source, authors from galaxy_clusters where id > ? order by id asc;

Feath Size: 300

 

galaxy_elements

select id, galaxy_cluster_id, g.key, value from galaxy_elements as g where id > ? order by id asc;

Feath Size: 1000

 

news

select date_created, id, title, message from news where id > ? order by id asc;

Feath Size: 300

 

objects

select id, event_id, name, 'meta-category', description, comment from objects where id > ? order by id asc;

Feath Size: 2000

 

tags

select id, name, colour from tags where id > ? order by id asc;

Feath Size: 300

 

taxonomies

select id, namespace, description from taxonomies where id > ? order by id asc;

Feath Size: 300

 

taxonomy_entries

select id, taxonomy_predicate_id, value, expanded from taxonomy_entries where id > ? order by id asc;

Feath Size: 300

 

warninglist_entries

select id, warninglist_id, value from warninglist_entries where id > ? order by id asc;

Feath Size: 10000

warninglists

select id, name, type, description from warninglists where id > ? order by id asc;

Feath Size: 300
