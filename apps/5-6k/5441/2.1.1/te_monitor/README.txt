Introduction
============
This add-on provides a method of visualizing and monitoring data generated by CounterFlow AI's ThreatEye NV product.
Data generated by a ThreatEye NV mediator can be pushed to this app by a supported ingest method (the Fluent Bit 
log processor and forwarder is recommended), allowing for drill down and alerting on generated data.

Requirements
============

A CounterFlow AI ThreatEye mediator instance with a supported forwarder is required for this application.

Installation
============

To install an add-on within Splunk Enterprise:

1. Log into Splunk Enterprise.
2. Next to the Apps menu, click the Manage Apps icon.
3. Click Install app from file.
4. In the Upload app dialog box, click Choose File.
5. Locate the .tgz file you just downloaded, then click Open or Choose.
6. Click Upload.


Configuration
=============

Initial configuration is needed to link correctly to the Counterflow ThreatEye NV mediator instance. 

First, create the required indices for data visualization. The indices should be named "te_event", te_flow", and "te_mgmt".

Go to "Settings" and select "Data Inputs" under the "Data" header.
1. Select "HTTP Event Collector"
2. Create three tokens, one for each of the respective data indices (they may already be generated, but it is recommended that you regenerate the token values for security purposes). Each token should be limited to its specific index. Please refer to Splunk documentation for further information on the HTTP Event Collector at (https://docs.splunk.com/Documentation/Splunk/latest/Data/UsetheHTTPEventCollector).
(3.) If using Fluent Bit, the token values are what you will use in conjunction with the server IP in order to push data to the proper indices on the Splunk instance. 

The ThreatEye Monitoring Suite for ThreatEye NV should now be ready to use, and will begin to populate with data one data is pushed to the instance.
