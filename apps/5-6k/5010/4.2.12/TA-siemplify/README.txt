# About Siemplify App

The Siemplify Splunk App allows Splunk Alerts to be sent to Siemplify and managed as Siemplify Alerts.

# REQUIREMENTS

* Splunk version 6.3 >=


# Topology and Setting up Splunk Environment


# Installation in Splunk Cloud

* Submit support ticket to Splunk for app to be installed.


# Installation of App

* This app can be installed through UI using "Manage Apps" or extract zip file directly into /opt/splunk/etc/apps/ folder.

The app must be setup prior to enabling any alerts actions.  The app requires 2 configuration parameters:
1) The URL of the Siemplify server.
2) The API Key to the Siemplify Server.  The API key can be generated via the Siemplify UI.  The user should login and go to Settings -> Advanced -> API Keys.  The API key needs to have admin privileges.

# Custom Alert Action

The Siemplify Splunk App will create a new action named "send_alert_to_siemplify"  It requires the following parameters:
Name: The value set here will affect the name of the Alert
Priority: The value set here will affect the priority of the Siemplify case.
Category: Used to define the visual family.
Environment: Maps to the environment in Siemplify. Leave blank for no environment. Templating with square brackets is supported.
Device Vendor: Used to define the vendor of the system sending the event into Siemplify. If the alerts were generated by Microsoft Sysmon use Microsoft or from a value within the alert/event using templating.
Device Product: Used to define the product of the system sending the event into Siemplify. If the alerts were generated by Microsoft Sysmon, this value should be Sysmon or from a value within the alert/event using templating.
Event Type: Used to define the event type in the Siemplify Event Configuration section. If the alert was looking for malicious processes, the event type should be something like “Process Found” or from a value within the alert/event using templating.
Time Field: Used to define the StartTime and EndTime of the Siemplify Case. If this is not supplied, it will check for the “_indextime” field. If it is unable to find “_indextime”, it will use the time the alert was generated. Templating is not supported.
Time Format: Used to define Time Field date format.
Expand MultiValue Fields: By setting this to 1 the system will find any multivalue fields and create additional fields mapping to each value in the multivalued field. For example, if a multivalue field, src_hosts, contains a value of: Server1, Server2, Server3. The system will create new fields of: src_hosts_0: Server1, src_hosts_1: Server2, src_hosts_2: Server3. This option is only supported when Bring All Events Data is disabled.
Bring All Events Data: This setting will attempt to bring the raw events that make up an alert containing a transforming command (chart, timechart, stats, top, rare, contingency, highlight). To support this a change to the Splunk Search Head is required.
