Deslicer AI Insights Add-on — Release Notes
============================================

Version 1.0.1 (2026-06-03)
---------------------------
Redesigned monitoring dashboard, refreshed branding, and a packaging fix.

Added:
  - Redesigned Insights Node Monitoring dashboard (Dashboard Studio) with two
    tabs:
      * Node Health — enrollment status, per-node status with a "last seen"
        indicator, collector error rate, and single-value counts for monitored
        hosts, events collected, and configuration files observed
      * Node Activity — configuration file-change activity, authentication and
        enrollment events, API key rotation events, and a node audit trail
  - Configuration file-changes panel and per-node "last seen" indicator

Changed:
  - Refreshed, on-brand Deslicer app icons
  - Unified dashboard status-color palette to match standard Splunk status colors
  - Dashboard index now admin-overridable via macros (deslicer_insights_index /
    deslicer_insights_internal) — works without edits for non-default indexes

Fixed:
  - Dashboard Studio packaging so the dashboard installs cleanly and passes
    AppInspect cloud vetting

Removed:
  - Legacy Simple XML version of the Insights Node Monitoring dashboard

Documentation:
  - Refreshed Splunkbase listing screenshots for the redesigned dashboard and
    current configuration/inputs screens
  - New companion walkthrough videos: a setup walkthrough (enrolling and
    configuring the add-on with the Deslicer Automation Copilot) and a DAP
    value-realization video (turning streamed configuration data into business
    insights and audited automation)

Version 1.0.0 (2026-03-xx)
---------------------------
Initial public release on Splunkbase.

New features:
  - Single app package for all Splunk Enterprise host tiers (indexers,
    search heads, heavy forwarders, cluster managers)
  - Collector binary for Linux x86_64 (amd64) and arm64 (aarch64)
  - Automatic enrollment with Deslicer Observer API using a one-time token
  - API key lifecycle management: expiry detection, automatic rotation,
    and revocation handling with re-enrollment blocking
  - Modular input supervisor with exponential backoff on binary crash
  - Deployment Server support: provision enrollment token via
    local/enrollment.conf before deploying to hosts
  - Configuration UI with encrypted credential storage (Splunk
    storage/passwords endpoint — no plain-text secrets in conf files)
  - Insights Node Monitoring dashboard (enable via local/app.conf on SHC)
  - Binary update detection: automatically restarts collector when a new
    binary is deployed via Deployment Server
  - Admin-only access: Configuration and Inputs restricted to admin and
    sc_admin roles; app hidden from non-admin users by default
