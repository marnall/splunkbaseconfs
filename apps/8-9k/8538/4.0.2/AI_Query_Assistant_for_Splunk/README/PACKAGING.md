# Packaging & Deployment — v4.0.0

This is the developer-facing checklist for cutting a Splunkbase release tarball
of **AI Query Assistant for Splunk** v4.0.0. End users do **not** read this
file — they install via Splunkbase or `splunk install app`.

## Build the tarball

```powershell
# From the app root (the directory that contains app.manifest)
python scripts/package_app.py
```

Output:

```
dist/AI_Query_Assistant_for_Splunk_v4.0.0.tar.gz   (≈4 MB)
```

The tarball is reproducible — successive runs with the same source produce
bit-identical bytes (we pin mtime to `SOURCE_DATE_EPOCH` or 2025-01-01).
Override the version with `--version 4.0.1` and the timestamp with
`--mtime <epoch>` if you need to.

### What is included

| Subtree | Purpose |
|---|---|
| `app.manifest`, `LICENSE` | top-level metadata |
| `default/` | app.conf, restmap.conf, collections.conf, views |
| `appserver/static/` | JS, CSS, icons |
| `bin/*.py` | REST handlers, install_deps.py, migrate_v3_to_v4.py |
| `bin/lib/agentic/` | Phase 1–3d agentic pipeline |
| `bin/lib/splunklib/` | vendored Splunk SDK (incl. `splunklib/ai/`) |
| `bin/lib/vendor/` | cryptography + cffi (v3 license verifier) |
| `bin/lib/prompts/` | NL→SPL system prompt |
| `bin/tools.py` | `@registry.tool()` local tools |
| `metadata/default.meta` | role/capability defaults |
| `README/` | release notes |
| `manifest.sha256` | per-file SHA256 + size, generated by the packager |

### What is excluded

| Subtree | Why |
|---|---|
| `bin/lib/site-packages/` | 150 MB pip-installed deps (langchain, pydantic, openai, anthropic, google-genai, mcp, …). Users populate this **post-install** by running `python bin/install_deps.py`. See `RELEASE_NOTES_v4.0.0.md`. |
| `scripts/` | dev artifacts (e2e tests, this packager, static check) |
| `local/` | per-instance overrides; never ship |
| `__pycache__/`, `*.pyc` | compiled-Python; AppInspect fails on these |
| `.git*`, `.vscode/`, `.idea/`, `.DS_Store` | VCS / editor metadata |
| AppInspect carve-outs: `openai/helpers/microphone.py`, `.keep` | already covered by site-packages exclusion, listed explicitly in the packager |

### What ships in `manifest.sha256`

Sorted, plaintext lines of `<sha256>  <size>  <relative path>`. One per file.
The manifest's *own* hash is not in the file (it can't reference itself); the
**tarball** SHA256 printed at the end of `package_app.py` is the value to
publish with the release.

## Pre-flight check

```powershell
python scripts/static_app_check.py --tarball dist/AI_Query_Assistant_for_Splunk_v4.0.0.tar.gz
```

This catches the most common AppInspect rejections **without** running the
real AppInspect tool:

- app.manifest / app.conf required fields, version coherence
- every `admin_external:*` handler declares `python.version`
- every Python file under `bin/` (excluding vendored splunklib / site-packages /
  vendor) compiles cleanly
- every SimpleXML view parses as well-formed XML
- KV collections declare `enforceTypes`
- bin/ Python sources don't contain `shell=True`, `os.system`, `verify=False`,
  raw `eval()/exec()` (vendored splunklib and bin/lib/vendor/ are exempt — we
  don't own that code)
- tarball doesn't contain `site-packages`, `__pycache__`, `*.pyc`, `.DS_Store`
- tarball is rooted at `AI_Query_Assistant_for_Splunk/` and includes
  `manifest.sha256`
- tarball is under 256 MB (warns over 100 MB)

Exit codes:
- `0` — clean
- `1` — at least one ERROR finding (must fix before AppInspect)
- `2` — only WARNINGS

## Run Splunk AppInspect

The static check is a fast precursor. Before publishing to Splunkbase, run
the real AppInspect at least once with the `cloud` and `manual` tag sets.

### Option A — local install (Linux / macOS / WSL)

`splunk-appinspect` depends on `python-magic` (libmagic), Pillow, and pywin32
on Windows. It is **not currently installable on Python 3.14**; the official
support matrix is Python 3.9–3.12. Pick whichever:

```bash
# In a Python 3.11 venv, on Linux/macOS or WSL
python3.11 -m venv .appinspect && source .appinspect/bin/activate
pip install splunk-appinspect
splunk-appinspect inspect dist/AI_Query_Assistant_for_Splunk_v4.0.0.tar.gz \
    --included-tags cloud --included-tags manual --mode test
```

### Option B — Splunk Cloud AppInspect API

For users on Python 3.13+/Windows-only environments, submit the tarball via
Splunk's cloud API (requires a [splunk.com](https://splunk.com) account):

```bash
TOKEN=$(curl -s -u "<splunkcom_user>:<splunkcom_pw>" \
    --data-urlencode "username=<splunkcom_user>" \
    --data-urlencode "password=<splunkcom_pw>" \
    -X GET https://api.splunk.com/2.0/rest/login/splunk \
    | jq -r '.data.token')

# Submit
curl -X POST \
    -H "Authorization: bearer $TOKEN" \
    -H "Cache-Control: no-cache" \
    -F "app_package=@dist/AI_Query_Assistant_for_Splunk_v4.0.0.tar.gz" \
    -F "included_tags=cloud" -F "included_tags=manual" \
    https://appinspect.splunk.com/v1/app/validate

# Poll the returned request_id every ~15 s until status=SUCCESS|FAILURE.
# Final report:
curl -H "Authorization: bearer $TOKEN" \
    https://appinspect.splunk.com/v1/app/report/<request_id>
```

### Option C — Docker

```bash
docker run --rm -v "$(pwd)/dist:/dist" \
    splunk/appinspect:latest \
    inspect /dist/AI_Query_Assistant_for_Splunk_v4.0.0.tar.gz \
    --included-tags cloud,manual
```

## Splunkbase upload

1. Sign in to <https://splunkbase.splunk.com/> with the Reallysec account.
2. App page → **Update App** → upload the tarball.
3. Paste the SHA256 printed by `package_app.py` into the release notes.
4. Splunkbase re-runs AppInspect Cloud automatically; resolve any new findings
   surfaced there even if Option A/B/C passed locally.
5. Wait for review (typically 1–3 business days for an existing app).

## Deploying to a Splunk instance (for testing)

```bash
# On the target Splunk server (10.2+, with Python 3.13 selected for the app)
sudo /opt/splunk/bin/splunk install app /tmp/AI_Query_Assistant_for_Splunk_v4.0.0.tar.gz
sudo /opt/splunk/bin/splunk cmd python \
    /opt/splunk/etc/apps/AI_Query_Assistant_for_Splunk/bin/install_deps.py
sudo /opt/splunk/bin/splunk restart
```

If installing on top of v3.0.7 data, run the conversation-thread backfill:

```bash
sudo /opt/splunk/bin/splunk cmd python \
    /opt/splunk/etc/apps/AI_Query_Assistant_for_Splunk/bin/migrate_v3_to_v4.py
```

## Release checklist

- [ ] `app.manifest` version updated and consistent with `default/app.conf` `[launcher] version`
- [ ] `README/RELEASE_NOTES_v4.0.0.md` reflects the actual change set
- [ ] `scripts/package_app.py` produces a tarball
- [ ] `scripts/static_app_check.py --tarball dist/…` exits 0
- [ ] AppInspect (Option A/B/C) reports no failures for `cloud` + `manual` tags
- [ ] E2E smoke tests pass against the target provider
- [ ] Splunkbase upload succeeds and the page lists the new version
