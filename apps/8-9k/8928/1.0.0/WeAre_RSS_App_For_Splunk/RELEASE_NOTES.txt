WeAre RSS App For Splunk — Release Notes

================================================================================
Version 1.0.0 — Initial Public Release
Release date: June 2026
================================================================================

Initial Public Release of WeAre RSS App For Splunk by WeAre Solutions.

New features

- RSS Feed Input modular input for collecting RSS 2.0 and ATOM feeds
- Splunk Universal Configuration Console (UCC) management UI for inputs and logging
- Configurable polling interval (10 seconds to 24 hours)
- Per-input settings for index, sourcetype, SSL verification, and HTTP timeout
- Checkpoint-based deduplication so only new feed entries are indexed
- Flexible event timestamp handling:
  - Indexing time (poll execution time)
  - Feed entry field (published, updated, or created)
- Optional Strip HTML tags setting to remove markup from title, summary, and content
- JSON event format with title, link, summary, content, authors, tags, and feed format
- Support for Splunk Enterprise and Splunk Cloud (Victoria)
- Apache License 2.0

Bug fixes

- Fixed Inputs page error on Splunk Python 3.9 caused by incompatible type hints in the modular input helper

Supported platforms

- Splunk Enterprise (standalone, distributed, search head clustering)
- Splunk Cloud
- Python 3.9 runtime on Splunk Cloud

Documentation

- Summary, usage instructions, and troubleshooting are included in README.txt

Known limitations

- Each input monitors a single feed URL
- Feed entries without a stable entry ID are not indexed
- Checkpoint history retains up to 10,000 entry IDs per input

Upgrade notes

- This is the initial public release. No upgrade path from a prior version applies.
