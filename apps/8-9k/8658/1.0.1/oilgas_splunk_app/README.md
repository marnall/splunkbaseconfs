# ABOUT THIS APP

* Oil and Gas is a Splunk app for monitoring oil and gas operational data, alarms, assets, and KPI health.
* The app includes prebuilt dashboards for asset overview, alarms, thresholds, KPI monitoring, onboarding, and asset management.
* The app includes the `OilGas` data model and searches that use it for dashboards, analytics, and KPI reporting.
* The app includes a scripted input that can generate sample oil and gas events from bundled CSV data for demonstrations and testing.
* The app includes CSV lookups for thresholds, alarms, well metadata, locations, test data, and prompt templates.
* The app uses Splunk KV store to store KPI definitions in the `kpi_store` collection.
* The app includes an optional AI chat custom visualization and a Splunk REST relay for external AI providers.
* The app stores AI secrets in Splunk secure password storage and does not expose the saved secret in the browser.
* Disclaimer: SoftServe does not access, receive, store, or process any information used by this app or generated by this app, including information submitted to or returned from the optional AI functionality. Such processing occurs only within the customer's Splunk environment and, if configured by the customer, the customer's selected AI provider.

* Version: 1.0.0

# REQUIREMENTS

* Splunk Enterprise with Python 3 support for scripted inputs and persistent REST handlers.
* A Splunk index for oil and gas events. The packaged content expects `index=oil_gas` unless you update the macros, searches, and scripted input configuration.
* Permissions to install apps and manage scripted inputs, saved searches, lookups, KV store, and password storage in Splunk.
* Oil and gas telemetry mapped to the `OilGas` data model, or use the bundled sample data generator for evaluation.
* Optional AI provider access if the AI feature is enabled. Remote endpoints must use HTTPS. Plain HTTP is supported only for local Ollama testing on `localhost`, `127.0.0.1`, or `::1`.
* Python version: python3

# RECOMMENDED SYSTEM CONFIGURATION

* Size the Splunk deployment according to expected event volume, dashboard concurrency, lookup activity, and data model acceleration requirements.
* Additional CPU, memory, and disk capacity should be planned if you enable data model acceleration, frequent scheduled searches, or large volumes of sample or production telemetry.
* Outbound internet access is required only when the optional AI feature is configured to use a remote provider.

# TOPOLOGY AND SETTING UP SPLUNK ENVIRONMENT

* This app can be deployed in two ways:

  1. **Standalone Mode**: Install the app on a single Splunk Enterprise instance that performs both indexing and searching for the app's content.

  2. **Distributed Deployment**: Install the app on the search head tier for dashboards, custom visualization assets, KPI management, and AI configuration. If you use the bundled scripted input for sample data generation, enable it only on the instance that is intended to run scripted inputs and forward events according to your standard Splunk deployment design.

* In a search head cluster, `ai_chat_settings.conf` is marked for search head cluster replication. AI secrets remain stored in Splunk secure password storage.

# INSTALLATION OF APP

* Download the packaged app archive and install it from Splunk Web using `Manage Apps > Install app from file`.
* Restart Splunk Enterprise if prompted after installation.
* Open the Oil and Gas app and review the `Onboarding Guide`.
* Install sample KPIs from the `Onboarding Guide` or `Setup KPI` view if you want demo KPI content.
* Leave the scripted input disabled unless you want to generate sample oil and gas events.
* If you build the release from source, run `npm install`, `npm run build`, and `npm run package`. The packaged archive is created as `dist/oilgas_splunk_app.tar.gz`.

# CONFIGURATION OF APP

## Initial setup

* Open the `Onboarding Guide` after installation.
* Review the expected fields for the `OilGas` data model and confirm your data source is mapped appropriately.
* If you do not use the default `oil_gas` index, update the app macros, scripted input stanza, and any saved searches that assume that index.

## Asset and lookup configuration

* Upload or update `oilgas_assets_wells.csv` if you want to provide your own well metadata lookup.
* Review the packaged lookups `well_locations.csv`, `well_test_data.csv`, `esp_thresholds.csv`, and `well_thresholds.csv` before production use.
* The thresholds editor updates `esp_thresholds.csv` and `well_thresholds.csv` using Splunk searches and `outputlookup`.

## KPI configuration

* Use the `Setup KPI` view to create KPI definitions.
* KPI definitions are stored in the Splunk KV store collection `kpi_store`.
* When a KPI is created from the setup page, the app also creates a corresponding saved search in Splunk.
* KPI searches must return a numeric field named `kpi_value`. They should also return `_time` and `is_latest` for full KPI visualization support.

## Alarm configuration

* The packaged searches `esp_alarms` and `well_alarms` run every 5 minutes and append results to `esp_alarms.csv` and `well_alarms.csv`.
* Review schedule, index scope, permissions, and lookup write behavior before enabling the app in production.

## Sample data generator

* The scripted input is defined in `default/inputs.conf` and is disabled by default.
* The generator reads bundled CSV files from `samples/wells_data/` and emits JSON events to stdout for Splunk ingestion.
* The scripted input writes to index `oil_gas`, sourcetype `_json`, unless you modify the stanza.
* The generator also supports historical backfill when run manually with `--start` and `--end`.

## Optional AI configuration

* Use the `AI Config` view to configure the provider protocol, endpoint URL, model, authentication behavior, TLS verification, timeout, and optional extra headers or request parameters.
* Supported provider protocols are OpenAI-compatible, Anthropic Messages, Google Gemini, and Ollama.
* Remote AI endpoints must use HTTPS. Plain HTTP is accepted only for loopback Ollama endpoints.
* The app stores connection settings in `ai_chat_settings.conf` and stores the API key or token in Splunk secure password storage.
* The browser UI sends requests to the app's Splunk REST endpoint, and the REST handler sends outbound requests only to the administrator-configured AI endpoint.
* SoftServe does not process any information used by or generated through the app or the optional AI feature.

# FILE ACCESS, PERSISTENCE, AND NETWORK USE

* The sample data generator reads bundled files from `samples/wells_data/`.
* The app writes generator logs to `$SPLUNK_HOME/var/log/oilgas_splunk_app/oil_gas_events.log`.
* The AI REST handler writes logs to `$SPLUNK_HOME/var/log/oilgas_splunk_app/oilgas_ai_rest.log`.
* KPI definitions are stored in the Splunk KV store collection `kpi_store`.
* Threshold and alarm workflows update or append to CSV lookups that ship with the app.
* The app does not initiate outbound network requests unless the optional AI feature is configured and used.
* When AI is enabled, outbound requests are sent only to the configured provider endpoint through the app REST relay.
* Manual historical backfill examples write redirected output to a local file such as `/tmp/backfill.txt` only when the administrator explicitly runs that command.

# TROUBLESHOOTING

* Confirm that `SPLUNK_HOME` is set correctly when running scripted inputs or manual backfill commands.
* If the sample generator does not produce events, verify that the target index exists and that the scripted input is enabled only on the intended Splunk instance.
* If dashboards are empty, verify that data is present in the target index, macros point to the correct index, and the `OilGas` data model is mapped and populated.
* If KPI pages fail to load or save data, verify that KV store is available and that the current role has permission to access app KV store data and saved searches.
* If threshold changes do not persist, verify permissions to run searches and update lookup files.
* If alarm lookups do not update, review the `esp_alarms` and `well_alarms` saved search schedules and permissions.
* For generator issues, review `$SPLUNK_HOME/var/log/oilgas_splunk_app/oil_gas_events.log`.
* For AI issues, review `$SPLUNK_HOME/var/log/oilgas_splunk_app/oilgas_ai_rest.log`.
* If AI connection tests fail, verify the configured endpoint URL, model, authentication settings, TLS options, and whether the selected provider protocol matches the target API.
* Remote AI endpoints must use HTTPS. Plain HTTP is supported only for local Ollama loopback testing.

# UNINSTALL & CLEANUP STEPS

* Remove `$SPLUNK_HOME/etc/apps/oilgas_splunk_app`.
* Remove `$SPLUNK_HOME/var/log/oilgas_splunk_app/oil_gas_events.log`.
* Remove `$SPLUNK_HOME/var/log/oilgas_splunk_app/oilgas_ai_rest.log`.
* Remove or archive any app-generated lookup data and KV store content if it is no longer required.
* Restart Splunk Enterprise to ensure the UI reflects the cleanup.
