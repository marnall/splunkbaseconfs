Innovators Toolkit — Privacy Notice
====================================

Summary
-------
This Splunk app stores small amounts of user-preference data in the user's
browser localStorage. No data is transmitted off the user's browser. No
third-party analytics, telemetry, or tracking is performed.

What is stored, where, and why
------------------------------
All entries below are stored in browser localStorage on the user's own
machine. The app never sends this data anywhere. Clearing browser storage
or using a private/incognito session removes it.

Every key name below was verified against the application source. Some
keys are per-dashboard: a base name is used and the current dashboard's
URL path is appended (non-alphanumeric characters replaced with "_"), so
one such key exists for each dashboard the user has visited.

Key                                  Purpose
----                                  -------
sit_ds_cloud_mode                    Whether the Design Studio is running
                                     in Splunk Cloud compatibility mode.
sit_ds_walkthrough_done              Tracks whether the first-run guided
                                     tour has been dismissed.
sit_ds_custom_templates              User-saved Design Studio templates.
                                     Stores the full dashboard XML/SPL of
                                     the user's saved designs.
sit_ds_version_history               Local version/undo history for the
                                     Design Studio. Stores full dashboard
                                     XML snapshots the user has saved.
sit-dark-light-mode                  Currently-selected dark or light mode.
sit-auto-refresh-interval            Auto-refresh interval the user picked.
sit-active-tab                       Last-active tab index for tab widgets.
sit-panel-sizes                      Custom panel layout widths from the
                                     drag-to-resize feature.
sit-collapsed-panels-<dashpath>      Panel collapse/accordion state. One
                                     key per dashboard ("<dashpath>" is the
                                     appended dashboard URL path).
sit-user-preferences                 Combined preferences blob written by
                                     the user-preferences-persist toggle
                                     (snapshot of the keys above).

Retention
---------
None of these keys have an expiry. They persist in browser localStorage
indefinitely until the user (or the in-app controls described below)
removes them. In particular, sit_ds_version_history and
sit_ds_custom_templates retain the full dashboard XML/SPL of the user's
saved designs indefinitely, with no automatic cleanup or size cap.

What is NOT collected
---------------------
- No usage analytics, page views, or click tracking.
- No data sent to third parties or the internet.
- No PII, credentials, or search results captured.
- No cookies set by this app (Splunk's own auth cookies are unrelated).

Server-side data
----------------
When a user clicks "Save" in the Design Studio (on-prem only), the
generated dashboard XML is written to the same Splunk instance via Splunk's
own REST API (/servicesNS/.../data/ui/views) under the user's existing
session. This is the standard Splunk dashboard save path and is governed
by the Splunk instance's own permissions and audit logging.

User control
------------
- Clearing the browser's localStorage for the Splunk Web origin (or using
  a private/incognito session) removes ALL of the keys above.
- The in-app controls only cover a subset:
  - The user-preferences-persist toggle's "Reset All" button removes
    sit-user-preferences, sit-dark-light-mode, sit-auto-refresh-interval,
    sit-active-tab, sit-panel-sizes, and the base "sit-collapsed-panels"
    name. Note: per-dashboard collapse keys are written with the
    sit-collapsed-panels-<dashpath> suffix, so "Reset All" does not
    necessarily clear those suffixed entries.
  - The Design Studio's "Clear All History" button removes
    sit_ds_version_history only.
  - There is NO in-app control that clears sit_ds_custom_templates,
    sit_ds_cloud_mode, or sit_ds_walkthrough_done; these persist until
    the user clears browser storage manually.
- Cloud Mode can be toggled on/off via the cloud-mode pill in the Design
  Studio top bar (this rewrites sit_ds_cloud_mode but does not remove it).
