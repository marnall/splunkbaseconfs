# TA-fwlog_eventgen

## Introduction

TA-fwlog_eventgen is a custom LOG generator by [Splunk EventGen](http://splunk.github.io/eventgen/).
This TA generates continuous event logs of general external Firewall facing internet.
It can be useful if we want a bunch of continuous sample logs.
Src and Dst IP address lists are from Eventgen App's samples.

## Installation

1. Install [Eventgen App](https://splunkbase.splunk.com/app/1924).
2. Enable the SA-Eventgen modinput by going to Settings > Data Inputs > SA-Eventgen and by clicking “enable” on the default modular input stanza.
3. Download & Copy/Move this App bundle into your ${SPLUNK_HOME}/etc/apps/ directory OR you can possibly install this App via Splunk WebUI normally.
4. Create `eventgen_fw` event index on Splunk WebUI.
5. Restart Splunk
6. Search by `index=eventgen_fw`
7. + Extract New Fields if you want.

Official installaration page of eventgen: http://splunk.github.io/eventgen/SETUP.html
We recommends "Install / Use Eventgen as a Splunk App".

## Description 

This TA generates it as csv format like below.
Src and Dst IP address lists are from Eventgen App's samples.
The src and dst port number and total communicated bytes are by random. 

```
index,host,source,sourcetype,_raw
eventgen_fw,eventgen,firewall_demo.csv,csv,"2022-10-15 10:29:34.107 action=ACCEPT src_ip=5.5.5.5 src_port=8922 protocol=TCP dst_ip=10.2.1.35 dst_port=80 bytes=489"
```

TA-fwlog_eventgen will generates 17 million events per 1 week.
You can adjust the log volume by modifying samples/firewall_demo.csv or eventgen.conf.
Refer to  http://splunk.github.io/eventgen/BASICS.html.


## Configuration (Copy of eventgen.conf)

[HOW TO READ](http://splunk.github.io/eventgen/CONFIGURE.html)

```
# Note, these samples assume you're installed as an app or a symbolic link in 
# $SPLUNK_HOME/etc/apps/eventgen.  If not, please change the paths below.

[firewall_demo.csv]
mode = sample 
sampletype = csv

timezone = local
earliest = -1m
latest = now

# Maximum number of events to generate per sample file. Defaults to -1. -1 means replay the entire sample. 
#count = -1

interval = 5
delay = 1

# Takes a JSON hash of 24 hours with float values to rate limit how many events we should see in a given hour.
hourOfDayRate = { "0": 0.30, "1": 0.10, "2": 0.05, "3": 0.10, "4": 0.15, "5": 0.25, "6": 0.35, "7": 0.50, "8": 0.60, "9": 0.65, "10": 0.70, "11": 0.75, "12": 0.77, "13": 0.80, "14": 0.82, "15": 0.85, "16": 0.87, "17": 0.90, "18": 0.95, "19": 1.0, "20": 0.85, "21": 0.70, "22": 0.60, "23": 0.45 }

# Takes a JSON hash of 7 days of the week in Splunk format (0 is Sunday)
dayOfWeekRate = { "0": 0.97, "1": 0.95, "2": 0.90, "3": 0.97, "4": 1.0, "5": 0.99, "6": 0.55 }

# Will randomize the number of events generated by percentage passed
# Example values: 0.2, 0.5
# Recommend passing 0.2 to give 20% randomization either way (plus or minus)
randomizeCount = 0.5

outputMode = splunkstream

# timestamp token
token.0.token = \d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}.\d{3}
token.0.replacementType = timestamp
token.0.replacement = %Y/%m/%d %H:%M:%S

# src_ip token
token.1.token = (5\.5\.5\.5)
token.1.replacementType = file
token.1.replacement = $SPLUNK_HOME/etc/apps/SA-Eventgen/samples/external_ips.sample
#token.1.replacementType = random
#token.1.replacement = ipv4

# dst_ip token
token.2.token = (10\.2\.1\.35)
token.2.replacementType = file
token.2.replacement = $SPLUNK_HOME/etc/apps/SA-Eventgen/samples/webhosts.sample

# src_port token
token.3.token = (8922)\s{1}
token.3.replacementType = random
token.3.replacement = integer[1025:65535]

# dst_port token TCP
token.4.token = (3389)\s{1}
token.4.replacementType = random
token.4.replacement = integer[1:65535]

# dst_port token UDP
token.5.token = (53)\s{1}
token.5.replacementType = random
token.5.replacement = integer[1:65535]

# bytes token
token.6.token = (489)\s{1}
token.6.replacementType = random
token.6.replacement = integer[1000:100000]
```
