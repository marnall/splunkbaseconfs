# CyberArk Audit for Splunk (for Linux 64-bit)

The CyberArk Audit app for Splunk fetches the events that are generated by the CyberArk services.
Customers that subscribe to these services can use the events to analyze and monitor their services in real time.

## Before you begin

In order to set up the Audit app, you must subscribe to the CyberArk auditing system.
After you subscribe, you'll receive the credentials and API configuration that you have to input as parameters to the CyberArk Audit app.

## Software

This app requires Python 3.13 and Splunk Enterprise/Cloud Platform 10.2 or higher. It runs on platform linux_x86_64

## Network (for Splunk Enterprise)

To fetch events, the Audit app authenticates and then communicates with the AWS API Gateway.
Therefore specific ports must be opened for outbound connectivity to the addresses listed below:


Dynamic configuration

| FQDN                                      | Port/protocol                      |
| ----------------------------------------  | ---------------------------------  |
| *.execute-api.{api-region}.amazonaws.com  | 443/HTTPS/TCP , and REST/API calls |
| {auth-endpoint}                            | 443/HTTPS/TCP , and REST/API calls |

* auth-endpoint and api-region are included in the parameters provided by CyberArk in order to configure the Audit app
