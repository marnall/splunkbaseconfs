Omega Core Audit for Oracle App for Splunk
by DATAPLUS
www.dataplus-al.com


1. REQUIREMENTS

1.1 Omega Core Audit

This app uses events generated by Omega Core Audit only. Omega Core Audit is a security software 
solution for Oracle databases. Omega Core Audit features access control, audit monitoring and 
protection in real-time; must be installed on the monitored Oracle Database[s].

To download Omega Core Audit, please visit:
https://dataplus-al.com/omega-core-audit

1.2 Splunk DB Connect

Required for "Pull" mode implementations only (see below).


2. HOW IT WORKS

Omega Core Audit deployment[s] generate security events in its own repository. The security events 
can be delivered to Splunk in two ways:

Push Mode - by Omega Core Audit LMS module. Events are sent in XML format using Splunk's TCP Input
Pull Mode - by Splunk. Events are retrieved by Splunk DB Connect using a SQL command


3. SETUP

3.1 App Install

Install online from SplunkBase, or from the downloaded file.

3.2 Index omega_ca

Create a Splunk index named omega_ca.

3.3 Data Input

Push mode - create the TCP Data Input. 
Pull mode - create the Splunk DB Connect Input

Templates for both input types are available at inputs_templates.conf.
Rename this file to inputs.conf. Make the same rename at /default/app.conf,
on "reload." under [triggers].
